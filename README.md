Project Description:

This project presents a comparative study between two CPU scheduling algorithms: Priority Scheduling and Shortest Remaining Time First (SRTF). The main objective is to analyze how different scheduling policies affect system performance, fairness, and process handling.
Priority Scheduling selects processes based on assigned priority values, making it suitable for policy-driven systems where certain tasks are more important than others. On the other hand, SRTF selects the process with the shortest remaining execution time, aiming to optimize efficiency and minimize waiting time.
This report evaluates both algorithms using multiple test scenarios, including cases of conflicting priorities, starvation-sensitive workloads, and validation cases. The comparison focuses on performance metrics such as Waiting Time (WT), Turnaround Time (TAT), and Response Time (RT), aiming to highlight the trade-offs between efficiency and fairness
